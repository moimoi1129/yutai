精霊クロス伝説 (Seirei Cross Densetsu)

優待クロス取引をテーマにした、ポップでかわいいUIの「艦これ」風シミュレーションゲーム。


1. コアコンセプト

テーマ: 株式投資の「優待クロス取引」×「精霊（擬人化キャラクター）」

ビジュアル: 白とピンクを基調としたポップで清潔感のあるデザイン

ゲーム性: クリッカー要素を排除。時間経過による「祈念（遠征）」、「労働」システム、戦略的な「出撃（権利日バトル）」を軸とする。
          キャラクターの「証拠金（HP）」、「疲労度（MP）」、「パケットポイント（PP）」を管理し、リソースを効率的に運用する。

3. 主要システム

A. ホーム ＆ 祈念 (Pray)

放置することで「資金」「通信パケット」「封筒」を調達する基本サイクル。

完了時に「現渡（げんわたし）！」ボタンで報酬を確定。

B. 封筒開封 (Gacha)

獲得した封筒（茶・白・金・虹）を開封。

演出として「現引（げんびき）！」などの投資用語を盛り込む。

C. 出撃 (Battle)

艦これライクなターン制・ウェーブ制バトル。

通信パケットを消費して進行し、ボス撃滅でマップ突破。

キャラクターの証拠金（HP）が20%以下になると「追証」状態（赤色ゲージ）となり、0%になると「退場」表記となる。
完全ロストではなく、議決権などの救済措置で復帰可能。

バトルにより疲労度（MP）が蓄積し、パケットポイント（PP）を消費する。

D. 補給 (Resupply)

キャラクターの回復・補給を行うセクション。

**労働システム**: 証拠金（HP）を時間経過で回復させる。艦これの入渠システムに相当。回復に必要なリソースは時間のみ（即時回復アイテムも検討中）。

**疲労度回復**: 優待アイテム（チョコレート等）を消費して、キャラクターの疲労度（MP）を回復させる。時間経過でも自然回復。

**PP回復**: 通信パケットを消費して、キャラクターのパケットポイント（PP）を回復させる。


E. 実績 ＆ ミッション

state.statistics に記録された累計データに基づき報酬を付与。

デイリー・期間限定ミッションによる継続的なプレイ要素。

4. UI 構成 (SPA: Single Page Application)

1200x720の固定解像度で、以下のタブを切り替えて進行する。

ホーム: 祈念・基本ステータス・チャットログ

優待精霊: キャラクター一覧・編成
    - キャラクターのステータスバー: 証拠金（HP）は緑色バー、疲労度（MP）はピンク色バー、パケットポイント（PP）はオレンジ色バーで表示。

優待品: アイテムインベントリ

封筒開封: ガチャ画面

補給: キャラクターの回復・補給を行う画面。

作戦: ミッションリスト

出撃: バトルマップ選択・進行

実績: 累計スタッツ・トロフィー

設定: システム設定

5. AI開発戦略 (Logic-First / Debug-First)

★コーディング注意点、要件定義をしっかり決めて、明示的な指示があってからコーディングを開始してください★

データ形式: 全てのマスターデータは .js ファイルで管理する。

管理手法: Googleスプレッドシートを正（ソース）とし、CSV経由でJSへ変換する。

AI指示: データ更新時は「CSVをJSの配列に変換」と指示する。

AIによるコード破壊を防ぎ、精度の高いコーディングを維持するための独自ルール。

フェーズ1: ロジック構築 (debug.html)

最優先: 見た目を無視し、debug.html 内で全てのロジックを完結させる。

ログ仕様: ログ出力は日本語翻訳せず、実行したソースコードをそのまま表示する。

例: systemLog("state.resources.money += 1500;");

視認性: 変数名（state.で始まる文字列）は自動的に青文字でハイライトする。

フェーズ2: コード分離 (main.js)

ロジックが安定した段階で、<script> セクションを main.js へ抽出する。

フェーズ3: UI接合 (index.html)

最終的に index.html (1200x720) を作成し、main.js と ID 紐付けを行う。

AIへの指示の掟

コンテキスト節約: CSSが巨大なファイルはAIに読み込ませない。

構造維持: 「ロジックのみを編集し、HTML構造は破壊しないでください」と常に念押しする。

変数名準拠: メッセージ表示は常に state.resources... のような内部変数名を表示させる。
